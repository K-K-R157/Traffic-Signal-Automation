PART A: DATA ACCESS LAYER (DAL) IMPLEMENTATION           

1. DATABASE DESIGN
The system uses MySQL database named "smart_traffic_db" with two core tables that handle user authentication and traffic violation management.

TABLE 1: app_users
Purpose: This table stores user account information for authentication and role-based access control in the system.

Column Details:
  - id : The unique identifier for each user. This number is automatically generated by the database when a new user is created.
  - username : The login name chosen by the user. This must be unique across all users to prevent duplicate accounts. Maximum length is 64 characters.
  - password_hash : The encrypted password stored using PBKDF2 hashing algorithm. We never store plain text passwords for security reasons. This field can hold up to 255 characters to accommodate the hash value.
  - role : The user's permission level in the system. Three roles are available:
    • SYSTEM_ADMIN - Full system access with all privileges
    • TRAFFIC_PERSONNEL - Can control signals and view violations
    • VIEW_ONLY - Can only view the system status
  - display_name : The friendly name shown in the user interface. For example, "Traffic Officer" or "System Administrator". Maximum 120 characters.
  - created_at : The date and time when the user account was created. This is automatically set by the database when a new record is inserted.


TABLE 2: traffic_violations
Purpose: This table records all traffic violations detected by the system during simulation. It helps track illegal lane changes and maintains a log for traffic personnel to take appropriate action.

Column Details:
  - id : The unique identifier for each violation record. Automatically generated by the database for every new violation logged.
  - vehicle_id: The license plate number of the violating vehicle. For example, "MH27K4521". This helps identify which vehicle committed the violation.
  - vehicle_type : The category of vehicle that violated traffic rules. Common types include "CAR", "TRUCK", "AMBULANCE". Maximum 16 characters.
  - side : The intersection side where the violation occurred. Possible values are "NORTH", "SOUTH", "EAST", or "WEST". This indicates the vehicle's approach direction.
  - violation_type : The type of violation committed. Currently, the system detects "SIDE_LANE" violations when vehicles illegally change lanes while crossing the intersection.
  - in_middle : A boolean flag (0 or 1) indicating if the vehicle was using the middle lane while entering the intersection. 1 means yes, 0 means no.
  - out_middle : A boolean flag (0 or 1) showing if the vehicle was using the middle lane while exiting the intersection. 1 means yes, 0 means no.
  - action_taken : A flag indicating whether traffic personnel have taken action on this violation. 0 means pending, 1 means action completed. Defaults to 0.
  - created_at : The exact date and time when the violation was detected and logged. Automatically set by the database.
