1.1 What is White Box Testing?
-------------------------------
White Box Testing is a testing technique where the tester has complete
knowledge of the internal code, structure, and logic of the application.
The tester examines the code to design test cases that cover all paths,
branches, and conditions.

Also known as:
- Glass Box Testing
- Clear Box Testing
- Structural Testing
- Code-Based Testing


1.2 White Box Testing Techniques Used
--------------------------------------

1. Statement Coverage
   - Execute every line of code at least once

2. Branch Coverage
   - Test all if/else conditions (true and false paths)

3. Path Coverage
   - Test all possible execution paths

4. Condition Coverage
   - Test all boolean expressions

SECTION 2: BLACK BOX TESTING
=============================

2.1 What is Black Box Testing?
-------------------------------
Black Box Testing is a testing technique where the tester evaluates the
software without knowing its internal code or structure. Testing is based
on requirements and specifications. The tester provides inputs and verifies
outputs without examining the code.

Also known as:
- Functional Testing
- Specification-Based Testing
- Behavioral Testing


2.2 Black Box Testing Techniques Used
--------------------------------------

1. Equivalence Partitioning
   - Divide input data into valid and invalid partitions

2. Boundary Value Analysis
   - Test values at boundaries and just beyond

3. Decision Table Testing
   - Test combinations of inputs and conditions

4. State Transition Testing
   - Test transitions between different states
